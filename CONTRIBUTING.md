# Contributing to Binary Atlas

Thank you for your interest in contributing to Binary Atlas.

This project is a personal learning and research project focused on static analysis of PE binaries. Contributions are welcome to improve detection capabilities, performance, and usability.

## How to Contribute

### 1. Fork the Repository
Create your own fork of the repository and clone it locally.

### 2. Create a Branch
Use a clear and descriptive branch name:
- feature/<feature-name>
- fix/<bug-name>
- improvement/<area>

Example:
feature/yara-enhancement

### 3. Make Changes
- Keep code clean and readable
- Follow existing project structure
- Add comments where necessary
- Avoid breaking existing functionality

### 4. Test Your Changes
Ensure your changes:
- Do not introduce errors
- Work with different PE samples where possible

### 5. Submit a Pull Request
- Clearly describe what you changed
- Explain why the change is useful
- Include examples or screenshots if applicable

---

## Contribution Ideas

You can contribute in many ways:

- Add new detection modules
- Improve YARA rule sets
- Enhance entropy or anomaly detection logic
- Optimize performance
- Improve report generation (HTML/TXT)
- Fix bugs or edge cases
- Improve documentation

---

## Code Style

- Use clear variable and function names
- Keep functions modular and simple
- Avoid overly complex logic when possible
- Follow Python best practices

---

## Reporting Issues

If you find a bug or issue:
- Open an issue with a clear description
- Include steps to reproduce
- Attach sample files if safe to share

---

## Note

This project is under active development and may contain incomplete features or bugs. Contributions should align with the goal of improving malware analysis workflows and learning.

---

Thank you for contributing.
